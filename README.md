# ℹ️ О приложении DigitalEducation

🌐 **Официальный сайт:** [https://digitaleducation.ru](https://digitaleducation.ru)  

**DigitalEducation** — это интерактивное приложение для обучения цифровой грамотности. Мы помогаем пользователям освоить работу с компьютером, файловой системой, операционной системой Windows и другими базовыми навыками. 

### Ключевые возможности:
- ✅ Готовые курсы по работе с файлами, папками и операционной системой.
- ✅ Создание собственных уроков с пошаговыми инструкциями.
- ✅ Поддержка компьютерного зрения для автоматической проверки действий пользователя (поиск элементов на экране).
- ✅ Визуальные подсказки (стрелки, рамки, подсветка, затемнение) для наглядного обучения.
- ✅ Отслеживание прогресса и статистика обучения.
- ✅ Светлая и тёмная темы.

Приложение разработано с использованием WPF и OpenCvSharp, ориентировано на пользователей, которые хотят повысить свою компьютерную грамотность в удобной интерактивной форме.

# 📚 Формат пользовательских уроков в DigitalEducation

Описание формата JSON-файлов, которые используются для хранения пользовательских уроков в приложении DigitalEducation.  
Файлы автоматически создаются через интерфейс (страница «Создать урок»), но вы также можете редактировать их вручную или генерировать программно.

---

## 📁 Расположение файлов

Пользовательские уроки хранятся в папке:  
`DataServices/Lessons/CustomLessons/` (относительно корня проекта или папки с приложением).  

Каждый урок — отдельный JSON-файл с именем вида `<ID урока>.json`.  
Все изображения (шаблоны для компьютерного зрения и подсказки) должны находиться в папке:  
`Learning/Engine/Templates/` (и могут быть организованы в подпапки).

> **Важно:** JSON-файлы должны быть в кодировке **UTF-8**.

---

## 🧱 Структура корневого объекта

```json
{
  "id": "string",
  "title": "string",
  "courseId": "string",
  "completionMessage": "string",
  "steps": [ ... ]
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `id` | string | да | Уникальный идентификатор урока. Генерируется автоматически при сохранении (например, `CustomLesson_20250226123045_abc123`). **Не изменяйте вручную**. |
| `title` | string | да | Название урока (отображается в списке и в шапке окна). |
| `courseId` | string | да | Идентификатор курса. Для пользовательских уроков всегда `"Custom"`. |
| `completionMessage` | string | да | Сообщение, которое увидит пользователь после успешного завершения урока. |
| `steps` | array | да | Массив объектов шагов (минимум один шаг). |

---

## 📝 Структура шага (`step`)

Каждый элемент массива `steps` — объект со следующими полями:

```json
{
  "title": "string",
  "description": "string",
  "hint": "string",
  "visionTarget": "string",
  "visionConfidence": 0.85,
  "requiresVisionValidation": true,
  "visionTargetFolder": "string",
  "requiredMatches": 1,
  "visionHint": "string",
  "hintType": "rectangle",
  "hintConfidence": 0.8
}
```

### Основные поля шага

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `title` | string | да | Заголовок шага (например, *"Шаг 1: Откройте рабочий стол"*). |
| `description` | string | да | Подробное описание действия, которое нужно выполнить. |
| `hint` | string | нет | Текстовая подсказка (отображается под описанием). |

### Поля для проверки через компьютерное зрение

Шаг может требовать обнаружения элемента на экране. Для этого используется **либо** одиночный шаблон (`visionTarget`), **либо** папка с несколькими шаблонами (`visionTargetFolder`). Если ни одно поле не задано, проверка не выполняется.

| Поле | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `visionTarget` | string | — | Имя файла шаблона **без расширения** (например, `CustomLesson_20250226123045_001`). Файл должен лежать в папке `Templates`. |
| `visionConfidence` | number | 0.85 | Порог уверенности (от 0 до 1). Чем выше, тем строже поиск. |
| `requiresVisionValidation` | boolean | `true`, если задан `visionTarget` или `visionTargetFolder` | Указывает, требуется ли проверка на этом шаге. |
| `visionTargetFolder` | string | — | Имя папки внутри `Templates`, содержащей несколько PNG-шаблонов. При проверке ищется любой из них до достижения `requiredMatches`. |
| `requiredMatches` | integer | 1 | Сколько элементов из папки должно быть найдено одновременно. |

### Поля для визуальной подсказки

Если шаг должен показывать на экране выделение (стрелку, рамку и т.п.), используются следующие поля:

| Поле | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `visionHint` | string | — | Имя файла шаблона **без расширения** для поиска места, где рисовать подсказку. |
| `hintType` | string | `"rectangle"` | Тип визуальной подсказки (см. таблицу ниже). |
| `hintConfidence` | number | 0.8 | Порог уверенности для поиска элемента подсказки. |

### Типы визуальных подсказок (`hintType`)

| Значение | Описание |
|----------|----------|
| `rectangle` | Пунктирный прямоугольник с подсветкой. |
| `arrow` | Стрелка, указывающая на элемент. |
| `highlight` | Жёлтое свечение (радиальный градиент). |
| `corner` | Оранжевый маркер в левом верхнем углу элемента. |
| `glow` | Двойное свечение (внешнее и внутреннее). |
| `dim` | Затемнение всего экрана, кроме области найденного элемента («дырка»). |

## 🖼️ Правила работы с изображениями

1. **Все изображения должны быть в формате PNG**.
2. При сохранении урока через интерфейс выбранные файлы копируются в папку `Learning/Engine/Templates/` и автоматически переименовываются:
   - Для `visionTarget`: `{id урока}_{номер:000}.png` (например, `CustomLesson_20250226123045_001.png`).
   - Для `visionHint`: `{id урока}_step{номер}_hint.png` (например, `CustomLesson_20250226123045_step1_hint.png`).
3. В JSON **указывается только имя файла без расширения** (например, `"visionTarget": "CustomLesson_20250226123045_001"`).
4. Если вы создаёте JSON вручную, поместите файлы изображений в папку `Templates` самостоятельно и пропишите их имена (без расширения) в соответствующих полях.
5. Для использования папки с несколькими шаблонами (`visionTargetFolder`) создайте подпапку внутри `Templates` и положите туда нужные PNG-файлы. Имена файлов могут быть любыми (в JSON они не указываются).

```json

  "title": "Создание папки на рабочем столе",
  "courseId": "Custom",
  "completionMessage": "Отлично! Вы научились создавать папку.",
  "steps": [
    {
      "title": "Шаг 1: Откройте рабочий стол",
      "description": "Убедитесь, что вы видите рабочий стол Windows.",
      "hint": "Нажмите Win+D, чтобы свернуть все окна.",
      "visionTarget": "CustomLesson_20250226123045_001",
      "visionConfidence": 0.85,

    },
    {
      "title": "Шаг 2: Нажмите правую кнопку мыши",

      "hint": "",
      "visionTargetFolder": "ContextMenuItems",
      "requiredMatches": 1,
      "visionConfidence": 0.8,

    }

```

## ✅ Рекомендации

- Для проверки используйте точные скриншоты элементов в натуральную величину.
- Порог уверенности (`visionConfidence`) подбирайте экспериментально: слишком низкий → ложные срабатывания, слишком высокий → элемент может не находиться из-за мелких отличий.
- Если на шаге не требуется проверка, просто не указывайте `visionTarget` и `visionTargetFolder`.
- Для `hintType = "dim"` изображение для подсказки (`visionHint`) не требуется — затемнение создаётся вокруг области, найденной по `visionTarget` (если задан) или на весь экран.

## ❓ Вопросы

**Можно ли использовать JPG вместо PNG?**  
Нет, движок компьютерного зрения ожидает именно PNG. Конвертируйте изображения перед использованием.

**Что будет, если в JSON указано имя файла, которого нет в папке Templates?**  
Проверка/подсказка не сработает, в логах появится ошибка. Убедитесь, что файл присутствует.

**Нужно ли перезапускать приложение после добавления нового урока вручную?**  
Да, реестр уроков загружается при запуске. Чтобы применить изменения без перезапуска, используйте интерфейс или вызовите `LessonRegistry.ReloadAllLessons()` программно.
